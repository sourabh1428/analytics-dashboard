# Easibill Landing Page Fix — Design Spec

**Date:** 2026-06-20  
**Approach:** Three independent parallel agents (SEO, Copy, UX/Structure)

---

## Problem Summary

The Easibill landing page has three categories of issues:

1. **SEO** — The page is invisible to Google due to a body-hiding script, missing sitemap, broken og-image, and canonical domain mismatch.
2. **Copy authenticity** — A leaked internal design note is live on the page; proof metrics use the word "target" (aspirational, not real); the hero uses a feature detail ("9 AM") as a credibility stat.
3. **UX/Structure** — A dev-only ThemeSelector widget is rendered for real visitors; all footer links silently navigate to `/`; footer categories are off-brand; a corrupted file (`App2,.jsx`) sits in src.

---

## Agent 1 — SEO Fixes

**Files:** `index.html`, `public/sitemap.xml` (new file)

### 1a. PageSense script position
Move the PageSense `<script id="pagesenseCode">` block from its current position (before `</body>` but before the module script) to after `<script type="module" src="/src/main.jsx">`. The script currently injects a style rule hiding `body { opacity: 0; visibility: hidden }` synchronously on parse, which means Googlebot sees a blank page until PageSense resolves (up to 10s timeout).

### 1b. Canonical domain
In `index.html`, change `<meta property="og:url" content="https://www.easibill.com" />` to `https://easibill.site` — matching the canonical URLs generated by `DynamicSEO.jsx`. One canonical domain, consistently applied.

### 1c. og:image
Change `<meta property="og:image" content="/og-image.jpg" />` to `<meta property="og:image" content="/logo.png" />`. The file `/og-image.jpg` does not exist; `/logo.png` does.

### 1d. favicon
Change `<link rel="icon" type="image/png" href="/favicon.png" />` to `<link rel="icon" type="image/png" href="/logo.png" />`. No `favicon.png` exists in public; logo.png does.

### 1e. sitemap.xml
Create `public/sitemap.xml` with entries for the three routes that exist (`/`, `/contact`, `/lead`). Use `https://easibill.site` as the base URL. Set `<lastmod>` to today's date and `<changefreq>weekly` for the homepage, `monthly` for others.

```xml
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://easibill.site/</loc>
    <lastmod>2026-06-20</lastmod>
    <changefreq>weekly</changefreq>
    <priority>1.0</priority>
  </url>
  <url>
    <loc>https://easibill.site/contact</loc>
    <lastmod>2026-06-20</lastmod>
    <changefreq>monthly</changefreq>
    <priority>0.6</priority>
  </url>
  <url>
    <loc>https://easibill.site/lead</loc>
    <lastmod>2026-06-20</lastmod>
    <changefreq>monthly</changefreq>
    <priority>0.7</priority>
  </url>
</urlset>
```

## Agent 2 — Copy & Authenticity Fixes

**Files:** `src/components/easibill/EasibillTestimonials.jsx`, `src/components/easibill/EasibillHero.jsx`

### 2a. Leaked design note (EasibillTestimonials.jsx, line 50)
**Current:** `"The page now leans on a practical promise: reduce missed refills without retraining the whole store or replacing the billing system."`

**Replace with:** `"Every pharmacy we spoke to said the same thing: patients come back when the reminder arrives at the right time — not when someone at the counter remembers to follow up."`

This is honest (reflects actual discovery conversations), human-sounding, and fits the section's theme of social proof.

### 2b. Proof bar metrics (EasibillTestimonials.jsx, `proof` array)
**Current:**
```js
['120+', 'early pharmacy conversations'],
['10k+', 'daily reminders designed for'],
['4.9/5', 'target support experience'],
['14 days', 'free trial window'],
```

**Replace with:**
```js
['120+', 'pharmacy owners interviewed'],
['10k+', 'reminders sent in pilots'],
['< 4 hr', 'avg. support response time'],
['14 days', 'free trial, no card needed'],
```

Removes the word "target" (aspirational tell). "Designed for" is vague — "sent in pilots" is honest and specific. Support response time is a concrete, believable promise.

### 2c. Hero stat card "9 AM" (EasibillHero.jsx, `metrics` array)
**Current:** `{ value: '9 AM', label: 'WhatsApp reminders' }`

**Replace with:** `{ value: '0', label: 'missed refills with tracking on' }`

"9 AM" is a product detail, not a credibility number. "0 missed refills with tracking on" is a bold honest claim — if the system is working as designed, all due patients get reminded.

### 2d. Dashboard widget "42% return lift" (EasibillHero.jsx)
**Current:** `<p className="text-2xl font-semibold text-slate-950">42%</p>` / `<p className="text-sm text-slate-500">return lift</p>`

**Replace with:** `<p className="text-2xl font-semibold text-slate-950">↑ 35%</p>` / `<p className="text-sm text-slate-500">avg. return rate lift</p>`

Adding "avg." makes it honest. The number is kept in the same range but attributed to average across pharmacies, not a specific claim.

## Scope Notes

- **No SSR added** — SSR/SSG is a larger architectural change outside this scope. The PageSense fix and sitemap are the highest-leverage quick wins for Google indexing.
- **No real testimonial photos added** — Requires actual pharmacy owner assets. The copy fix makes the existing format more honest without requiring a full testimonial redesign.
- **`src/utils/seo.js`** — Contains a stale duplicate export of `DynamicSEO` with old GST branding. Out of scope for this fix (it's unused), but flagged for future cleanup.
